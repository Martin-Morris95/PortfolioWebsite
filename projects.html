Panoramic Viewer

[C#] [.Net] [Visual Studio] [MVC] [Pannellum] [HTML/CSS/JS]

Features

  • Creation of Panoramic Viewer
  • Hosted link for the viewer that can be viewed externally by clients
  • Optional settings for branding, floor plan and waypoints
  • Control panel to view and copy IFrame code for embedding

Pannellum is a lightweight, free, and open source panorama viewer for the web. Built using HTML5, CSS3, JavaScript, and WebGL, it is plug-in free. (https://pannellum.org/)

I was tasked to create a user-friendly workflow to allow employees to create interactive panoramic walkthrough by leveraging the power of the open source software Pannellum. The current workflow would mean manually altering code to place way points and to target the 360-degree render output images of the projects.

I thought this process would be too cumbersome for our small IT team, so I set out to create a web application to generate the code automatically. The web application I created allowed employees to upload there renders and allow them to place waypoints to other rooms. I then extended the experience by allowing a floor plan quick travel option.

The experiences then are assigned a unique link and hosted on the company’s web server allowing employees to send out the links for marketing purposes or to their employees to embed on their websites as the tool also allows easy generation of Iframe code.

Help Desk Notes Tool

[C#] [.Net] [Visual Studio] [WPF] [XAML] [SQL] [LINQ]

Features

  • Upload work related job notes to database
  • Word template generation for new notes to ensure harmonious design/style
  • Search functionality with tags and title
  • Caters for multiple users
  • Compiles all notes in one easy to access location

This was the first application I made upon starting my new job as an IT Support Officer. The idea came about when I was being trained alongside another employee. As you could expect each of us had many questions and we would write down the answers in our notebooks. This became messy quickly and made finding the answers to problems a lengthy process. We also began asking the same questions repeatedly as we both worked different days.

I realised we needed an efficient way to compile our knowledge, allow us to share notes, tips and tricks whilst storing them securely so the tool can be used by future new employees giving the company an easier way to train my role for the future.

I created the application in Visual studio using C# and WPF. The application allows users to generate a note template (which opens word automatically), they then add searchable tags and a description and then upload it to the database. All the IT team can then open the app and search for that note.

System Info Tool

[C#] [.Net] [Visual Studio] [WPF] [XAML] [SQL] [UI] [UX]

Features

  • Tracks computers performance
  • Gathers information about each computer in the company and stores it in a Database
  • Detects when a users password is about to expire and sends them a personalised email 7 and 3 days prior. (Also takes into consideration if this day falls on a weekend and will warn you earlier otherwise)
  • Warns users of timed events (temp drive erase, password expiry, computer uptime, time sheet status)
  • Allows for quick creation of a helpdesk ticket.
  • Update users outlook signature, launch timesheet portal, reveal utility account password and view system information
Disclaimer: I only worked on the front end of this project. All data gathering logic, utility password generation and signature update was done prior to my contribution to the project

The system info tool was initialy created to gather data for each computer user, RAM, GPU, CPU, etc. This allowed IT to compile a computer list that we could access and instantly know who was using which machine and its specs. The tool also displayed realtime CPU, RAM, and storage performance which allowed the support team to quickly see if a computer had any performance issues.

After sometime working at the company that had created and deployed this tool I noticed that no one paid attention to the tool and I believed we could rework the front end to be more pleasent and engaging for employees whilst also maintaing the functionality the tool was intended for.

I started off with removing all the data that the employees wouldnt want to see/care about and moved it to its own seperate window (see image 2). This freed up alot of space and made the tool less cluttered. I then emphasized important information such as the IP address, computer name and current user name as these are three things we mostly ask for when offering remote help via the phone. I then reworked the alert system banner (see Image 3) to be more obvious and eye catching. This is an attempt to stop users missing there password resets and temp drive erase dates. Lastly I added a whole new section labeled IT Help Desk. This section allows for an automated email template to be generated to send helpdesk tickets and also includes the help desks contact number. This was done in an attempt to lower the amount of direct calls the head of IT recieves for trivial helpdesk level jobs.

As our employees tend to work remotely/out in the field expired passwords can become quite a hassle if they have not reset them prior to the lock out date. As this happens quite alot I incorporated a password expiry checker that will alert the user via an automated email 7 and 3 days prior to it expiring with instructions on how to do so. This along with the new and improved alert banner system I hope to lower the password lockout cases.

Revit Library Addin

[C#] [.Net] [Visual Studio] [Windows Forms] [SQL] [LINQ] [Revit API]

Features

  • Revit Addin with various tools
  • Tool 1: Revit Library - Allows users to import family models into revit
    • Search by name, category or both
    • Request model upload to Database managed by Revit Admins
    • Request Audit of models to Revit Admins
    • Easily see family details with quick info tab upon clicking
    • Multiple display options

Travel Request Tool

[C#] [.Net] [Visual Studio] [MVC] [SQL] [HTML/JS/CSS]

Features

  • Company Travel Request Form - Allows employees to request travel >> Manager approves >> Trip is sent to booking admin
  • Area 1: Request Travel
    • This page is avaliable to all employees
    • Allows users to fill out travel details for proposed trip
  • Area 2: Approve Travel
    • This page is only visable to managers
    • Allows managers/team leaders to view requests and approve/deny them
  • Area 3: Booking Admin
    • This page is only visable to the IT team
    • Allows IT to see all the booking data in an organised fashion
  • Area 4: Booking Admin
    • This page is only visable to employees who are responsible for booking the work related trips
    • Allows booking employees to easily see what trips need to be booked

This project was created to solve a problem we found in the workplace. Employees would request work trips via email. This meant the only records of the trip requests were easily lost, not approved correctly or simply fogotten about. We also faced the issue with a break down of communication between booking staff as emails were sent to individuals and not a group inbox.

I worked along side one of the booking staff to create this web application to ensure it was tailored to their need. We decided to seperate the application into several layers. All employees could request travel this would then send an email their manager which would prompt them to view the approve Travel Section. The managers then had the choice to either accept or deny the trip. If denied the process stops here and the employee recieves an email with the reason for denial. If approved an email is sent to the booking employees prompting them to view the Booking Admin section. Here the staff have access to all the details needed to book the trip. Once booked they mark it as booked and the employee recieves a confirmation email.

This was an extremly fun app to build. The entire request travel section is dynamic and transforms and prompts the employee depending on the input. The deployment of the app also streamlined the entire process and allowed for data analytics as now all travel requests were being tracked and stored.
